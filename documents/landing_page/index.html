Core Capabilities

Your keys. Your server. Your rules.

KONKIN is a local MCP server that bridges AI agents with your self-hosted crypto wallet daemons — with every transaction requiring your explicit approval.

🔒

Fully Self-Hosted

Runs entirely on your machine. No cloud, no custody, no trust — just you, your keys, and your wallet daemons.

👤

Human-in-the-Loop

Every transaction queues for your approval via Telegram, WhatsApp, Discord, SMS, or a local web UI. The agent waits.

⚖️

Configurable Quorum

Set up 1-of-N approval (any channel approves), N-of-M multi-sig-style control, or custom auto-rules by amount, time, or volume.

📋

Audit Log

You decide what gets recorded. Full history of agent requests, approvals, and denials — stored locally and owned by you.

🧩

Modular Wallet Support

Bitcoin, Monero, and Litecoin today — with a growing list of coins added as dedicated wallet modules over time.

🔌

MCP Native

Works with Claude Desktop, Cursor, and any MCP-compatible AI client out of the box via stdio, SSE, or Streamable HTTP.

Live Preview

See it in action

From mobile approval notifications to a full audit trail — KONKIN keeps you in control at every step.

Mobile Approval Notification

Your AI agent requests a transaction — you receive an instant prompt to approve or deny from any device.

Multi-Channel Authentication

Configure approvals across Telegram, WhatsApp, Discord, SMS, a REST webhook, or via another MCP agent.

Per-Coin Approval Queue

Each coin maintains its own queue. Pending requests wait for your quorum before the agent can proceed.

Immutable Audit Log

A full local record of every request, decision, and outcome. You own the data — and you decide what's logged.

Architecture

Simple by design

A single JAR, one config file. KONKIN sits between your AI client and your wallet daemons — intercepting every request for your approval.

┌─────────────────────────────────────┐ │ AI HOST / CLIENT │ │ (Claude Desktop, Cursor, …) │ └──────────────┬──────────────────────┘ │ stdio / SSE / Streamable HTTP ▼ ┌─────────────────────────────────────┐ │ KONKIN MCP SERVER │ │ │ │ ┌──────────────────────────────┐ │ │ │ Human-in-the-Loop Gate │ │ │ │ per-coin queue · quorum │ │ │ │ Telegram / WhatsApp / │ │ │ │ Discord / REST / MCP Agent │ │ │ └──────────────────────────────┘ │ │ │ │ ┌────┐ ┌────┐ ┌────┐ + future… │ │ │BTC │ │XMR │ │LTC │ │ │ └────┘ └────┘ └────┘ │ │ │ │ Config: config.toml │ │ Artifact: konkin-server.jar │ └─────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────┐ │ YOUR WALLET DAEMONS (local) │ │ bitcoind · monero-wallet-rpc · … │ └─────────────────────────────────────┘

One command to start: java -jar konkin-server.jar config.toml
